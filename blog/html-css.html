CSS display: inline vs inline-block

Imagine you are trying to file away your collection of books. You start with an empty bookcase. For lack of a better plan, you start at the top shelf and start filling it from left to right. When the shelf is full, you move down a shelf and start filling again from left to right. Repeat until there are no more books left. This is the way web browsers display HTML elements by default, in the absence of any styling rules, and it is called normal flow. The HTML elements are the books and the bookcase is whatever HTML container you’re trying to fill. HTML elements that behave in this way, filling up space by positioning themselves next to each other until they run out of room, are called inline elements. Examples of inline elements are: <a> <img> <code> <span>

Some books might contain very little content but they will take up the entire width of the bookshelf, forcing the other books to fill up from the next shelf down. Maybe it’s your special hardcover edition of Fifty Shades of Grey, signed by the author. HTML elements that behave in this way, starting on a new line and taking up the entire width of the container, are called block elements. Examples of block elements are: <div> <li> <h1> <table>

There is another difference between inline and block elements besides the way they are laid out on the page, and that is their size. Inline elements take up only as much space as their contents. Think of the way a plastic shopping bag collapses or expands based on what you put inside. Specifying a height or width for an inline element will have no effect. Block elements, as we have already seen, take up the width of the container by default. You can set height and width for a block element but even if the width is less than the container, the element that comes after it will still be forced to a new line.

The behaviour of inline and block elements can be changed from their default simply by setting the element’s display property to inline or block. There is a third display option that is a hybrid of inline and block and it is called, quite appropriately, inline-block. With inline-block, an element will position itself side-by-side with adjacent elements but it will respect a height or width property.

In summary, the three display options differ with regard to two things: how they handle an element’s positioning relative to adjacent elements, and whether they allow an element’s size to be set. This handy table will hopefully help you to remember the differences.

Positioning Size
inline Adjacent to neighbouring elements until the width of the container is filled, then wraps to a new line Based on size of contents, cannot be set
inline‑block Adjacent to neighbouring elements until the width of the container is filled, then wraps to a new line Can be set
block Starts on a new line, and the following element is forced to a new line as well Can be set
